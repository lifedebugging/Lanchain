Architecture

FastAPI + uvicorn keeps Janus alive as a persistent server
Client is a separate script — server and client are decoupled
reload=True in uvicorn requires string reference "filename:app", not the app object

Streaming

AgentExecutor.astream() is step-level, not token-level — one chunk per agent step
astream_events(version="v2") gives token-level streaming with tool support
Filter for event["event"] == "on_chat_model_stream" to get tokens
Extract token with event["data"]["chunk"].text
Yield tokens with \n appended so iter_lines() splits correctly on client
iter_lines() waits for \n before yielding — design around this
print(token, end="", flush=True) for inline token printing

Generators

async def + yield = async generator
Can't await a dict or string — only coroutines
@retry_with_backoff() breaks async generators — don't decorate generators with it
Return the generator object, don't await it — return response_func() not return await response_func()

AgentExecutor chunk structure

astream() yields {"model": {"messages": [...]}} — not {"output": ...} as docs suggest
Always verify chunk structure with a print before assuming keys
